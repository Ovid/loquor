## Executive Summary

The branch adds Georgian (`ka`) as a Phase-1 **output-only** language (display translated, commands typed/raw-sent in English; no input lexicon, no LLM fallback for `ka`), plus a one-retry model-download path and an fr/de/es lexicon parity pass. Overall quality is high: `tsc -b` is clean, the full suite passes (1143 tests), and the headline narrowing work (`lang`/`lexLang` split, `OutLang` type, `PipelineRefs` regrouping, `ABSTAIN` relocation) all integrates correctly. No crashes or data-loss bugs were found. The highest-impact finding — flagged independently by three specialists and confirmed against the code — is a **suppressed-download-modal state leak**: picking Georgian latches the hook's `modalOpen` flag true (masked only at render), and switching away re-surfaces an unexpected focus-trapping WebLLM download modal. Two findings are genuine a11y bugs (Important); the rest are narrow logic gaps and safety-net suggestions.

## Critical Issues

None found. (F1 is a real, reproducible a11y/UX bug but requires a specific switch-away sequence rather than occurring on load, so it is rated Important.)

## Important Issues

### [I1] Suppressed download-modal state leak — switching away from Georgian pops an unexpected focus-trapping modal

- **File:** `src/llm/useModelDownload.ts:286-308` (setLanguage) + `src/ui/Terminal.tsx:194` (`upgradeModalOpen`)
- **Bug:** `setLanguage('ka')` runs the normal no-model path and calls `setModalOpen(true)` (when the model isn't cached and the player hasn't declined). Terminal hides it via `upgradeModalOpen = nl.modalOpen && !outputOnly`, so the player never sees or dismisses it — but the hook's `modalOpen` stays `true`. No switch-away path clears it: the `'off'` branch, the installed early-return, and `abortInFlight` all leave `modalOpen` untouched. Switching **ka → off / en / an already-installed fr-de-es** flips `outputOnly` to false, making `upgradeModalOpen` become `true` → the WebLLM download/upgrade modal opens with no user action, trapping focus and marking the background `inert`.
- **Impact:** An unexpected focus-trapping modal that offers a multi-GB download the player never requested. Per CLAUDE.md, an unexpected keyboard/SR focus trap is a hard a11y bug, not a polish item.
- **Suggested fix:** Don't open the modal for output-only languages in the first place — guard the `setModalOpen(true)` with `!OUTPUT_ONLY_LANGS.has(lang)` in `setLanguage`. This also clears the latent stale-flag state and guarantees the documented model-download egress stays unreachable for `ka`.
- **Confidence:** High
- **Found by:** Logic & Correctness, Concurrency & State, Security & A11y (3 specialists; two reproduced it with unit tests)

### [I2] English landing examples/how-to voiced with Georgian phonemes

- **File:** `src/ui/Landing.tsx:111-154` (plate carries `lang="ka"`; examples region inherits it) + `src/ui/landingExamples.ts` (`ka` examples are English) + `src/ui/landingStrings.ts:170-172` (`ka.howToBody` embeds English `„open the mailbox"`)
- **Bug:** For `ka`, the landing command examples are English strings (read-Georgian / type-English by design) but render inside a region that inherits `lang="ka"` from the plate, with no `lang="en"` wrapper. A screen reader voices English text with Georgian phonemes. Same issue for the inline English command quoted in the Georgian how-to body.
- **Impact:** WCAG 2.2 §3.1.2 (Language of Parts) violation — degraded/garbled SR pronunciation for exactly the population a11y serves. The project already does this correctly for the in-game beta notice (`Terminal.tsx:271-280` splits `lang="ka"`/`lang="en"`), so this is an inconsistency the landing missed.
- **Suggested fix:** Wrap the English example list in `lang="en"` (set `lang="en"` on the examples region when `exampleLang === 'ka'`), and wrap the inline `open the mailbox` in `<span lang="en">` in `howToBody`.
- **Confidence:** Medium-High
- **Found by:** Security & A11y

### [I3] Mid-drain switch to `ka` can invoke the input LLM under `activeLang='ka'`

- **File:** `src/llm/translatePipeline.ts:~904` (drain guard) + stage-7 `generateRaw` at `~357`; reachability via `src/llm/useNaturalLanguage.ts:163-167` (epoch)
- **Bug:** The compound-drain guard breaks only when `phase !== 'on'`. A language switch does **not** bump `epochRef` (epoch bumps on vocab/game change only), so switching from a full-model language (fr) to `ka` mid-drain does not abandon the in-flight queue. Subsequent clauses resolve `activeLang='ka'`, `lex=null` (no ka lexicon), and `grammarOnly=false` (model still `full`), falling through to stage 7: `generateRaw(buildPrompt(clause, ctx, vocab, 'ka'), grammar)`. The new `FEWSHOTS['ka'] ?? FEWSHOTS.en` fallback prevents a crash and silently masks it — an input-LLM inference that `ka` is specified to never perform.
- **Impact:** Bounded (the LLM emits a canonical English command, not Georgian; requires a clause queued at the instant of the switch), but it violates the `ka` "no input LLM, raw-send only" invariant.
- **Suggested fix:** Extend the drain abort condition to also bail when the active language is in `OUTPUT_ONLY_LANGS`, abandoning the queue the same way the `phase !== 'on'` branch does.
- **Confidence:** Medium
- **Found by:** Logic & Correctness

### [I4] (Player-experience — talk-to-me-first) Georgian on Zork II/III shows 100% English with no in-game cue

- **File:** `src/ui/Terminal.tsx:175` (`showBetaNotice` correctly gated off when no ka corpus) + `Landing.tsx:206-209` (English-only badge)
- **What the player sees:** A player who selects "ქართული (beta)" via the **in-game picker** on Zork II/III (which have no `ka` corpus) gets a fully English game, English raw-send input, and no in-game indication of why. The honest "English only" badge only appears on the Landing plate, which a mid-session in-game switcher never sees.
- **The product decision:** Phase-1 Georgian is Zork-I-only output translation; II/III have no corpus and degrade silently to English.
- **Reasoning (from spec/CLAUDE.md):** Georgian is corpus-only with no LLM fallback (small models can't produce Georgian); only Zork I has a corpus.
- **Override offer:** Per CLAUDE.md's "player experience overrides product decisions" rule, flagging for your call, Ovid — e.g. an in-game status note when `ka` is active but the running game has no `ka` corpus (mirroring `showBetaNotice`'s gate, inverted). Negligible if you consider the Landing badge + "(beta)" label sufficient, but it's your decision to make, not a silent pass.
- **Confidence:** Medium
- **Found by:** Error Handling & Edge Cases (verified)

## Suggestions

- **[S1] `notices.ts:24` / `prompt.ts:81-83` — Partial widening drops the compile-time completeness check for fr/de/es.** `ByLang`/`FEWSHOTS` now require only `en`; a future entry omitting e.g. German compiles and silently ships English instead of failing the build. Intentional for `ka` (dead input path), but it also relaxed the fr/de/es guarantee. Fix: `Record<LexLang | 'en', T> & Partial<Record<'ka', T>>` to keep fr/de/es mandatory while leaving `ka` optional, or add a string-inventory test. _(Error Handling)_
- **[S2] `useModelDownload.ts:247-250` — orphaned F-8 retry timer.** The retry timer reuses the shared `stallTimerRef`; `requestDownload` supersedes via `abortRef.current?.abort()` only (not `abortInFlight`), so a supersede while a retry is pending leaves it to fire and null out the new download's stall-watchdog handle (no rogue load — `stale()` guards it — but the new watchdog ref is lost/un-cancellable). Low reachability (picker/modal are `inert` during `downloading`). Fix: route `requestDownload` supersede through `abortInFlight`, or guard the retry callback's null-out behind `!stale()`. _(Concurrency)_
- **[S3] `types.ts:19` & `corpus/index.ts:15` — `OUTPUT_ONLY_LANGS` and `CORPUS_ONLY_LANGS` are two independent `Set(['ka'])` with no test pinning the invariant.** They can drift in Phase 2. Add a one-line test asserting `[...CORPUS_ONLY_LANGS].every(l => OUTPUT_ONLY_LANGS.has(l))` (subset, not equality). _(Contract & Integration, Error Handling — 2 specialists)_
- **[S4] CLAUDE.md still describes the NL/output layer as "EN/FR/DE/ES".** The "a fix in one language is a fix in all" section should note `ka`'s output-only asymmetry so a future contributor doesn't wire `ka` into the input path. _(Security & A11y)_
