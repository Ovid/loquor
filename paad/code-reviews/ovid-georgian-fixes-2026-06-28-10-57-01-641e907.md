## Important Issues

### [I1] Spanish (`es`) cyclops scare-word has no `ulises`/`odiseo` alias — Spanish players can't use their own word for the puzzle

- **File:** `src/llm/lexicon/es.core.ts` (absent entry); cf. `src/llm/lexicon/fr.core.ts:248` (`ulysse: 'ulysses'`) and the `ka` aliases added on this branch at `src/llm/lexicon/ka.core.ts:207,213` (`ოდისევსი: 'odysseus'`, `ულისე: 'ulysses'`)
- **Bug:** This branch added the Latin `ულისე → ulysses` alias for `ka`, and `fr` already maps `ulysse → ulysses`. But `es.core.ts` has **no** scare-word alias at all. The English/Latin spellings `odysseus`/`ulysses` raw-send via the vocab passthrough (`zork1.vocab.ts:27,396`), and German `Odysseus` is spelled identically to English so it passes through by coincidence — but Spanish `Ulises`/`Odiseo` are spelled differently, so a Spanish player typing their own word misses to the LLM. `fr.core.ts:245-246` documents that this exact un-aliased path makes the LLM "return 'look' and leave the Cyclops blocking the stairs."
- **Impact:** A Spanish player who reaches the cyclops and types the natural Spanish name (`Ulises`/`Odiseo`) gets the cyclops staying put, blocking the only staircase up — a puzzle stall solvable only by knowing to type the English/Latin form. Behind the LLM net (so not a hard `ka`-style dead-end), and pre-existing rather than introduced here, but it is the textbook "fixing German while Spanish is broken for the identical reason" pattern CLAUDE.md names.
- **Suggested fix:** Add `ulises: 'ulysses'` and `odiseo: 'odysseus'` to the `verbs` table in `es.core.ts` (mirroring `fr.core.ts:248`). Optionally add a `de` alias for symmetry, though `de` is functionally covered by the identical `Odysseus` spelling. Out of this branch's stated Georgian scope — flag for a follow-up rather than necessarily folding in here.
- **Confidence:** High (gap verified by grep: `fr` has it, `de`/`es` do not; passthrough confirmed)
- **Found by:** Contract & Integration
