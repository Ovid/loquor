## Suggestions

- **Scroll fix has no automated guard (accepted, documented).** jsdom has no
  layout engine and the stack has no Playwright, so the scroll-safe overlay/modal
  behavior is verifiable only by the spec's manual checklist (320 / 375 / 520px +
  a short landscape window, both themes). A CSS refactor can silently reintroduce
  top-clipping with every Vitest test green. Re-run the checklist on any touch to
  `src/ui/landing.css` / `src/ui/components.css`. (This is a known, spec-accepted
  gap — listed for completeness, not a defect.)
- **Title-box / corner-control vertical share at ≤520px (informational, no fix).**
  `.title` spans the full content width as a block; the absolutely-positioned
  dismiss ✕ (top/left 14px) and theme toggle (top/right 18px) overlap its *box*
  vertically. At 320px the centered "Loquor" glyphs (≈150–165px at the 2.5rem
  clamp floor, within the ~254px content box) leave ~45px clearance each side, so
  there is no visual collision today. Re-check clearance if the wordmark text or
  `letter-spacing` ever changes. (a11y agent, confidence 60.)
