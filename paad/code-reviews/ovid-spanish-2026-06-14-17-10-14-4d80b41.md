## Critical Issues

### [C1] Spanish contractions `a+el→al` / `de+el→del` never composed — masculine objects render `a el …` / `de el …`

- **File:** `src/translate/corpus/zork1.es.templates.ts` (out-sides at lines 60, 67, 185, 189, 197, 256, 260, 283, 287, 293, 313, 317, 321, 325, 329, 360, 374, 382, 406, 410, 438, 462, 528, 550, 554, 558, 690) and `src/translate/corpus/zork1.es.objects.ts` (no object carries the `alDef`/`delDef` keys the header at line 3 advertises).
- **Bug:** `match.ts` substitutes `{obj.def}` with the literal `def` string ("el ladrón", "el espejo", …) and has no contraction logic. So any template with `a`/`de` immediately before a `{obj.def}`/`{obj2.def}` slot produces `a el …` / `de el …` for masculine-singular objects. Verified at runtime:
  - `"Attacking the thief is pointless."` → `"Atacar a el ladrón no sirve de nada."` (want `al ladrón`)
  - `"The thief contains:"` → `"Dentro de el ladrón hay:"` (want `del ladrón`)
  - `"There's nothing special about the mirror."` → `"Nada especial en cuanto a el espejo."` (want `al espejo`)
- **Impact:** Pervasive, user-visible grammatical breakage across the most common command classes (combat against el trol / el ladrón / el cíclope, container listings on el saco / el buzón / el ataúd / el baúl / el bote, `examine`/`look behind` on el espejo / el cuadro / el muro / el altar). Affects 66 of 129 objects. It ships silently: all gates pass, and the file header (line 6–10) falsely asserts "NEVER put «de» or «a» immediately before a slot," so a casual reader believes the hazard is handled.
- **Suggested fix:** Pick ONE strategy and make code + comment + spec agree:
  - **(a) Spec's design (recommended, data-only):** add `alDef`/`delDef` per-object form keys to every object a contraction-prone template can bind (e.g. `delDef: 'del ladrón'`, `alDef: 'al ladrón'`; feminine → `'de la botella'`/`'a la botella'`), and change the ~26 template out-sides to `{obj.delDef}` / `{obj.alDef}`. Gender lives in the per-object string, exactly as `def`/`indef` already do.
  - **(b) French discipline (what the header claims):** actually rephrase those templates to avoid `de`/`a` before the slot — but note a literal `del`/`al` + `{obj.bare}` only works for masculine objects, so this fails for feminine bindings; (a) is the robust choice.
  - Either way, update the templates header and reconcile with design §2.5/§3.3.
- **Confidence:** High (reproduced at runtime; converged on by all three specialists)
- **Found by:** Spanish-correctness, Contract & Integration, and pre-dispatch inspection

## Important Issues

### [I1] `catches fire … you were holding it` template mis-agrees in number for plural light sources

- **File:** `src/translate/corpus/zork1.es.templates.ts:340-343`
- **Bug:** `out: '{obj.def} se prende fuego. Por desgracia, la sostenías en ese momento.'` hardcodes feminine-singular `la sostenías` (and singular `se prende`). Correct for `la lámpara`/`la antorcha`, but if bound to the plural `las velas` (candles, a held light source) it yields `Las velas se prende fuego … la sostenías` — wrong number twice (`se prenden` / `las sostenías`).
- **Impact:** Narrow/off-path (held candles burning) but a genuine grammar error. Same class as the plural-agreement hazard the header explicitly warns about.
- **Suggested fix:** Pin the candle/torch/lantern variants as full strings in `zork1.es.strings.ts` (the project already pins other agreement-sensitive light-source lines), or restrict the template to singular-feminine bindings.
- **Confidence:** Medium
- **Found by:** Spanish-correctness
