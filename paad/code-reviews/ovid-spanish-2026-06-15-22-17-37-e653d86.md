## Important Issues

### [I1] French Loud Room echo re-voices with the elided article intact
- **File:** `src/translate/loudEcho.ts:39-45`
- **Bug:** `loudEchoToken` takes the last whitespace token and strips leading
  quotes/inverted marks (`/^[¡¿"'«»]+/`) and trailing punctuation, but does **not**
  strip a leading article or French elision. For a French clause whose last token is
  an elided noun (`prends l'or`, `l'épée`, `d'or`), the stored player word is `l'or`
  (the leading-quote strip only removes a *leading* `'`, not the mid-token `'` after
  `l`). The Loud Room echo then renders `` `${d} ${d} ...` `` =
  **`l'or l'or ...`** instead of `or or ...`.
- **Impact:** Player-visible wrong word in the French Loud Room (a "player-perfect"
  echo concern, the same class as the earlier I1 echo work). Spanish is mostly
  unaffected because its lexicon player words already drop the article
  (`['bar','barra']`).
- **Suggested fix:** In `loudEchoToken`, after isolating the last token, strip a
  leading French elision (`^[ld]'`) and/or a leading article, mirroring
  `isForeignNoun`'s head-stripping in `inputTranslate.ts`. Add a French elided-noun
  case to `loudEcho.test.ts` (current tests only cover bare nouns and leading/
  trailing marks) and an end-to-end assertion in `pipeline.uat.test.tsx` (which today
  only checks the full source string is recorded, never `loudEchoToken`'s reduction).
- **Confidence:** Medium-High (80)
- **Found by:** Error Handling & Edge Cases; confirmed by Verifier

## Suggestions

- **[S1] revoice-map collision on a shared canonical last word** — `src/translate/loudEcho.ts:51-59`, `src/ui/Terminal.tsx:55-65`. `echoMap` is keyed by `loudEchoToken(canonical)`, so two clauses with the same canonical (`coge la barra` and `agarra barra` → `take bar` → key `bar`) overwrite; last writer wins. Low impact — same canonical ⇒ same echo word; only the surface synonym differs. (conf 62)
- **[S2] `withContractions` uses case-exact `startsWith('el ')`** — `src/translate/corpus/zork1.es.objects.ts:454`. Latent only: every current `def` is lowercase, but a future `'El …'` or leading-space `def` would silently produce ungrammatical `de el …`/`a el …`, and the UAT gate's hardcoded inputs wouldn't catch it. Normalize/trim before the prefix check. (conf 65)
- **[S3] `nounSet` rebuilt on every `fillElidedVerbs` call** — `src/llm/inputTranslate.ts:156` does `new Set(Object.values(nouns).flat())` per input line, while siblings `vocabVerbHeads` and `VOCAB_WORD_SETS` cache via `WeakMap<Vocab,Set>`. Use a `WeakMap<NounLexicon,Set>` to match the idiom. Negligible perf, purely consistency. (conf 65)
