## Executive Summary

The natural-language layer (English → grammar-constrained Zork command via on-device
WebLLM) is well-structured, thoroughly tested (84 NL tests pass, `tsc` clean), and faithful
to its design spec. No crash- or corruption-on-every-turn defect went unmitigated by tests,
with **one exception (H)**. The highest-severity finding is **architectural/privacy (T):**
enabling NL fetches multi-GB model weights and executable WASM from third-party CDNs
(HuggingFace, raw.githubusercontent.com) with no integrity pinning — materially at odds with
the app's documented "fully client-side / offline / privacy" promise. Five Important findings
total; the rest are hardening suggestions. Overall confidence: high — findings were
cross-verified against the actual source, and several plausible-sounding reports (parse
brittleness, char-prompt routing) were correctly rejected as unreachable given grammar
constraints and the game's protocol.

## Important Issues

### [I1] NL model download fetches from third-party CDNs with no integrity pinning (privacy-promise violation)
- **File:** `src/llm/engine.webllm.ts:34-35` (and `:79`)
- **Bug:** `CreateMLCEngine(this.modelId, { initProgressCallback })` passes **no `appConfig`**, so WebLLM falls back to its built-in `prebuiltAppConfig`, fetching model weights from `huggingface.co` and the model-lib **WASM** from `raw.githubusercontent.com` at runtime. No SRI / integrity verification.
- **Impact:** Contradicts CLAUDE.md's "fully client-side / offline / privacy" posture and the no-CDN principle; the modal copy "runs entirely on your device" (`ModelDownloadModal.tsx:28-29`) understates the one-time third-party download. The model-lib WASM executes in the page origin, so a tampered/MITM'd artifact is arbitrary in-origin code execution. First-run also leaks fact-of-use (IP + feature) to those hosts.
- **Suggested fix:** Pin an explicit `appConfig` with self-hosted (or integrity-checked) `model` + `model_lib` URLs under `public/`; update the modal copy to disclose the one-time third-party download; document the network egress in CLAUDE.md. (At minimum, disclose; ideally self-host to keep the offline promise true.)
- **Confidence:** High
- **Found by:** Security
- **Note:** Rated Important rather than Critical — gated behind explicit user opt-in, HTTPS transport, no execution without consent — but it breaks an explicit documented promise, so it tops the Important list.

### [I2] Download cancel race can re-enable NL and persist `enabled:true` against the player's cancel
- **File:** `src/llm/useNaturalLanguage.ts:104-131` (`.then` at 114-117, `cancelDownload` at 128)
- **Bug:** `cancelDownload` aborts the controller then sets `phase:'off'`, but the load `.then` has **no guard** against an aborted/superseded controller. `WebLlmEngine.load` only checks `signal.aborted` at entry and after `create()` resolves; `FakeLlmEngine.load` has no post-create check. If `create()` resolves on/around the abort tick, `.then` runs `setInternal({phase:'on'})` + `writeNlPref({enabled:true})`, clobbering `off`→`on` and persisting enabled against the cancel.
- **Impact:** Player cancels a download but NL silently turns on and is persisted; re-enables/re-prompts on next reload.
- **Suggested fix:** Capture the `AbortController` in the closure; in both `.then` and `.catch` bail when `ac.signal.aborted` or `abortRef.current !== ac`. Also have the fake engine re-check `signal.aborted` after its progress loop.
- **Confidence:** High
- **Found by:** Logic & Correctness + Concurrency & State (agreement)

### [I3] `nl-source` (player's English) line is corrupted by the following append-style VM echo
- **File:** `src/glkote-react/reduce.ts:62-71` and `177-185`
- **Bug:** After `translate` runs `echoLocal(english)`, the nl-source line is the last buffer line. A real VM input echo arrives with `append:true` (see reduce.ts:36-41; reduce.test.ts:112). Because the nl-source text is the English sentence (not bare `>`), it misses the bare-`>` branch (62) and hits the generic merge (65): text becomes `english + commandEcho` and `input` flips true. At line 178 the `p.input` short-circuit then reclassifies the line to `'input'` — the nl-source preservation at line 180 is never reached.
- **Impact:** On every NL-translated turn (with the realistic append-style echo), the English source line is corrupted (e.g. `go north` + `north`) and loses its nl-source styling. Existing `bridge.test.ts:162-167` does **not** catch this because it feeds a non-append echo.
- **Suggested fix:** In `bufferParagraphs`, when `last.prev?.kind === 'nl-source'` (or `last` is nl-source), push the appended input echo as a NEW line rather than merging. Add a test with an `append:true` echo immediately following `echoLocal`.
- **Confidence:** High
- **Found by:** Logic & Correctness

### [I4] Watchdog timeout leaves the in-flight `generate()` orphaned (no cancellation)
- **File:** `src/llm/useNaturalLanguage.ts:154-204`; `src/llm/types.ts:31`
- **Bug:** `LlmEngine.generate(prompt, grammar)` has **no `AbortSignal`**. On watchdog timeout the `Promise.race` rejects but the underlying WebGPU inference runs to completion, orphaned. No in-hook concurrent-translate guard (no short-circuit on `pending`).
- **Impact:** A timed-out (or superseded) generation keeps consuming GPU; its result is discarded. The `disabled={nl.pending}` on `CommandInput` (Terminal.tsx:134) prevents a second *user* submit, so the concurrency hole is mostly closed at the UI — but the orphaned inference and inability to cancel remain (each timeout can strand another generation).
- **Suggested fix:** Add an optional `AbortSignal` to `generate`; abort the inference when the watchdog fires. Keep the disabled-input mitigation.
- **Confidence:** High
- **Found by:** Error Handling + Concurrency & State (agreement)

### [I5] `writeNlPref` silently resets the other field when stored prefs are corrupt
- **File:** `src/llm/nlpref.ts:25-33`
- **Bug:** `writeNlPref` merges `patch` onto `readNlPref(store)`. If stored JSON is corrupt/unreadable, `readNlPref` returns `DEFAULT` (`enabled:false, declined:false`), so a partial write (e.g. `writeNlPref({declined:true})`) silently **clobbers a previously-persisted `enabled:true`**.
- **Impact:** Corruption of one field resets the other to default on the next write — player loses their enabled choice with no signal.
- **Suggested fix:** Preserve unknown/extra keys on merge; log a parse failure instead of silently falling back. (Acceptable as best-effort, but it is a silent data reset.)
- **Confidence:** Medium
- **Found by:** Error Handling & Edge Cases

## Suggestions

- **[S1] WebGPU model never `unload()`ed** — `src/ui/Terminal.tsx:39`. `WebLlmEngine` is created once and never unloaded; unmount cleanup (64-68) disposes only the ZMachine. Story changes do NOT remount Terminal (no `key` in `App.tsx`), so the model is reused (fine); leak only manifests on a genuine Terminal unmount. Add an unmount cleanup calling `llmEngine.unload()`. *(Concurrency)*
- **[S2] `'__UNKNOWN__'` abstain sentinel duplicated** across `translate.ts:5`, `engine.webllm.ts:59`, and the grammars — export a shared `ABSTAIN` constant. *(Error Handling)*
- **[S3] Triplicated percent calc** `Math.round((loaded/total)*100)` in `NlToggle.tsx:28-29`, `ModelDownloadModal.tsx:18-21`, plus normalization in `engine.webllm.ts:38-42` — extract a `pct(loaded,total)` helper. *(Contract)*
- **[S4] No test pins `grammarForSignature(realSig) === GBNF`** (`grammar/index.test.ts`) — signatures are correct today but unguarded; drift silently disables NL. Add a real-signature → GBNF mapping test. *(Contract)*
- **[S5] Watchdog vs error discrimination via `err.message === 'watchdog'`** (`useNaturalLanguage.ts:181-190`) — fragile for non-Error rejections; both branches `sendLine(english)` so impact is only the notice text. Use a sentinel error class/symbol. *(Error Handling)*
- **[S6] `isCached` probe effect re-runs on every phase change** (`useNaturalLanguage.ts:67-83`) — confirmed NO infinite loop / improper re-enable (the `phase==='off'` guard holds); just redundant async churn with unordered `setInstalled` resolves. Narrow deps / split the auto-enable into its own effect. *(4 specialists, agreement)*
- **[S7] `Math.min(nav.deviceMemory, 8)` is a dead clamp** (`capability.ts:61-63`) — `deviceMemory` is spec-capped at 8; only `< 4` matters. Drop the `Math.min`. *(Logic)*
- **[S8] `probe-error` branch has no diagnostic** (`capability.ts:70-73`) — a real WebGPU probe exception is indistinguishable from no-GPU when debugging. `console.warn` the caught error. *(Error Handling)*
- **[S9] `getContext` is an unmemoized fresh closure** (`Terminal.tsx:91`) defeating `translate`'s `useCallback`; `viewRef` lags one commit (stale window closed in practice since translate is user-initiated). Optionally `useCallback` getContext. *(Concurrency)*
- **[S10] Dead `onTurn` seam / doc drift** — `onTurn` (bridge.ts:67,107; ZMachineOptions) is never passed by Terminal; CLAUDE.md says the LLM intercepts at the bridge, but NL actually intercepts at `CommandInput.onSubmit`. Remove or document the unused seam; reconcile the comment. *(Concurrency)*
- **[S11] Silent input drop if `engineRef.current` is null** (`Terminal.tsx:92-93`) — optional-chained `echoLocal`/`sendLine` no-op the turn; window is practically unreachable (engine set synchronously, input disabled until a line request). Optionally surface a notice. *(Contract)*
- **[S12] Prompt-injection surface, bounded** (`prompt.ts:35-40`) — game text + English interpolated with no delimiting, but output is grammar-constrained into the sandboxed Z-machine, so worst case is a wrong-but-valid command (no RCE). Optional delimiting of untrusted segments. *(Security)*
- **[S13] Bare-`>` filter drops a pathological `nl-source` of `>`** (`Scrollback.tsx:24`) — filter on kind so only VM-emitted bare prompts are dropped. *(Logic)*
